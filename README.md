# 外部排序算法的实现

## 一、什么是外部排序

当待排序的数据量太大，无法一次性全部载入内存时，就需要使用外部排序。它的核心思想是分而治之：先把大文件拆成小块，在内存里分别排序并输出为临时文件，最后再通过合并机制将这些文件组合成一个全局有序的大文件。

外部排序通常分为两个阶段：

- 第一阶段（生成初始归并段）：根据k的大小，假设内存中最多同时容纳k个元素，使用"置换-选择"或等价的分块排序方法生成若干个有序的初始归并段（run），写入磁盘。
- 第二阶段（k路归并）：先根据"最佳归并树"确定归并方案，再使用k路平衡归并分若干趟合并这些归并段，直到得到完整的有序输出文件。k在两阶段中都作为统一控制参数使用。

## 二、你的任务

你需要实现external_sort.cpp中的两个核心接口，补全外部排序的完整逻辑：

### 2.1 GenerateInitialRuns（第一阶段：生成初始归并段）

```cpp
std::vector<std::string> GenerateInitialRuns(std::ifstream &input, int k);
```

**参数说明：**

- `input`：已打开的输入文件流，内含 SNS 个整数，每行一个。
- `k`：统一控制参数。第一阶段假设内存中最多同时容纳k个元素，第二阶段则作为归并路数上限。
- 返回值：一个`std::vector<std::string>`，内含你创建的所有归并段文件路径（各文件内的整数必须已按从小到大排序，且所有归并段都应非空）。

### 2.2 KWayMerge（第二阶段：k路归并）

```cpp
std::string KWayMerge(const std::vector<std::string>& run_files, int k);
```

**参数说明：**

- `run_files`：第一阶段生成的归并段文件列表。
- `k`：每一趟归并的路数上限。
- 返回值：最终排序输出文件的路径（通常固定命名为`sorted_output.txt`）。
